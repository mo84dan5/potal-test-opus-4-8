# FN-015 イベント

## 概要
NPC等をタップすると始まる**スクリプト化イベント**。例: 話しかけると主人公が実際に歩いて案内される、
岩が動いて道が開く。イベント中は**見回し(視点回転)以外の操作は不能**。

## イベントの構成
イベントは `EventStep` の列(`GameEvent.steps`)。`Interactable.event` に持たせ、タップで開始する。

| ステップ | 内容 |
| --- | --- |
| `say` | 一定時間メッセージを表示(`eventMessage`)。タップ送りではない |
| `walkTo` | 主人公を (x,z) まで自動で歩かせる(目的地方向へ `desiredVelocity` を設定。地形追従・衝突は移動処理が担当)。到達 or タイムアウトで次へ |
| `moveProp` | 可動プロップ(`EventProp`)を (toX,toZ) へ duration 秒かけて smoothstep 補間。コライダーが位置追従するので道が開く |
| `wait` | 一定時間待つ |

## 進行と操作制限
- `EventService.tick(session, dt)` が `GameSession.activeEvent`(`ActiveEvent`)の現ステップを進める。
- メインループは `activeEvent` 中、`applyStick`/タップ/ダッシュ/滑空を呼ばず `EventService` が駆動する。
  入力アダプタのコールバックも `onLook`(見回し)以外はイベント中ガードする。
- `TickUseCase` はイベント中ポータル通過判定をスキップ(自動歩行の誤通過防止)。
- 移動は既存 `MovementService` を再利用(walkTo は目標速度を置くだけ)。`walkTo` は到達できない場合の
  ソフトロック防止に 12 秒のタイムアウトを持つ。

## 可動プロップ
- `World.props`(`EventProp`)。`TickUseCase` の衝突解決にプロップのコライダー(現在位置から導出)を含める。
- 描画は `ThreeRendererAdapter` が `propMeshes` を毎フレーム位置反映(`syncProps`)。

## デモ(昼の世界)
- 「案内役」(6,3): タップ → 主人公を崖の前 (12,12) まで歩いて案内(`day-guide`)。
- 「岩どかし」(-2,9): タップ → 岩 `day-rock` が (1,7) へ動いて道が開く(`day-rock`)。

## 設計(SOLID / クリーンアーキテクチャ)
- domain/values `EventScript`(EventStep/GameEvent/ActiveEvent)・entities `EventProp`・services `EventService`
  は three.js/DOM 非依存。`EventService` は単一責務(イベント進行)で `MovementService` にのみ依存。
- 非イベント時の挙動(会話/扉/ポータル/移動)は不変。

## テスト観点
- EventService: say/wait の時間進行、walkTo の desiredVelocity 設定と到達完了、moveProp の補間と完了、複数ステップ消化と終了。
- EventProp: コライダーが現在位置(y=0)を反映。
