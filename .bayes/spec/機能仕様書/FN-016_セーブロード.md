# FN-016 セーブ/ロード

## 概要
ゲーム状態(ワールド/プレイヤー姿勢/フラグ/完了イベント/可動プロップ位置)を**文字列のセーブコード**として
保存・復元する。HUDの「💾 セーブ」からパネルを開き、コードをコピーして保存、貼り付けて「ロード」で復元する。
将来は同じセーブコードを**画像**に内包する形式へ拡張予定。

## 状態(GameSnapshot)
保存対象は `GameSnapshot`(プレーンなデータ):
- `version` / `worldId` / `player{x,y,z,yaw,pitch}` / `flags` / `completedEvents` / `props{id→{x,z}}`。
- 一時状態(進行中イベント・会話)は保存せず、ロード時に解除する。

## 設計(SOLID / 安全に忘れるクリーンアーキ)
境界を「**スナップショット ⇄ 文字列(セーブコード)**」に固定する。

| 層 | 要素 | 責務 |
| --- | --- | --- |
| domain/values | `GameSnapshot`, `SnapshotCodec`(ポートIF), `isValidSnapshot` | 保存データの形・変換契約 |
| domain/services | `SaveService.capture/restore` | 集約 `GameSession` の状態の採取/復元(単一責務・プラットフォーム非依存) |
| adapter/persistence | `Base64JsonSnapshotCodec` | ポート実装。`PW1.`+base64(encodeURIComponent(JSON))。base64 等の具象をここに閉じ込める(DIP) |
| main / index.html | セーブ/ロードパネル | クリップボード(コピー/貼り付け)。イベント中・パネル表示中は見回し以外の操作をロック |

- domain は base64 などの**文字列化の実装を知らない**(`SnapshotCodec` ポートのみ)。差し替え可能(DIP)。
- セーブコード(文字列)を不変の契約にすることで、**将来の「画像で保存/ロード」は、同じ文字列を画像へ
  詰める/取り出す新アダプタを足すだけ**で済む(OCP)。`SaveService`/`GameSnapshot`/codec は変更不要=安全に忘れられる。
- ロードは未知ワールドID等で例外(改ざん検知)。UI は例外メッセージを表示する。

## 操作
- 「💾 セーブ」→ パネルにセーブコードを表示。「コピー」でクリップボードへ。
- コードを貼り付け→「ロード」で復元(成功でパネルを閉じる/失敗で理由を表示)。
- イベント中はセーブボタンを隠す。パネル表示中は移動/タップを止める(見回しは可)。

## 関連コード
- `src/domain/values/GameSnapshot.ts` / `src/domain/services/SaveService.ts`
- `src/adapters/persistence/Base64JsonSnapshotCodec.ts`
- `src/main.ts`(配線・パネルUI・ロック) / `index.html`(パネル)

## テスト観点
- SaveService: capture が状態を採取 / restore が復元しイベント解除 / ラウンドトリップ / 未知ワールドで例外。
- Base64JsonSnapshotCodec: encode→decode 一致 / 空白許容 / 接頭辞なし・復号不可・不正内容で例外。
