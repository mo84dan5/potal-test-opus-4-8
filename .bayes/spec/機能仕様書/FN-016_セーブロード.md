# FN-016 セーブ/ロード

## 概要
ゲーム状態(ワールド/プレイヤー姿勢/フラグ/完了イベント/可動プロップ位置)を**文字列のセーブコード**として
保存・復元する。HUDの「💾 セーブ」からパネルを開き、コードをコピーして保存、貼り付けて「ロード」で復元する。
将来は同じセーブコードを**画像**に内包する形式へ拡張予定。

## 状態(GameSnapshot)
保存対象は `GameSnapshot`(プレーンなデータ):
- `version` / `appName` / `savedAt` / `worldId` / `player{x,y,z,yaw,pitch}` / `flags` / `completedEvents` / `props{id→{x,z}}`。
- `appName`(アプリ名)・`savedAt`(保存日時 ISO 8601)は v2 で追加。**画像のキャプションにも使う**。
  - 時刻・アプリ名は外部都合(非決定的)なので、`SaveService` に `appName` と `Clock`(時刻ポート)を
    注入する(DIP)。サービスは `new Date()` を直接呼ばず、テストで決定的に検証できる。
  - 旧 v1 コード(これらが無い)も読めるよう、`isValidSnapshot` は両フィールドを「string か未定義」で許容する。
- 一時状態(進行中イベント・会話)は保存せず、ロード時に解除する。

## 設計(SOLID / 安全に忘れるクリーンアーキ)
境界を「**スナップショット ⇄ 文字列(セーブコード)**」に固定する。

| 層 | 要素 | 責務 |
| --- | --- | --- |
| domain/values | `GameSnapshot`, `SnapshotCodec`(ポートIF), `isValidSnapshot`, `Clock`(時刻ポート) | 保存データの形・変換契約・時刻供給 |
| domain/services | `SaveService.capture/restore` | 集約 `GameSession` の状態の採取/復元(単一責務・プラットフォーム非依存。`appName`/`Clock` 注入) |
| adapter/persistence | `Base64JsonSnapshotCodec` | ポート実装。`PW1.`+base64(encodeURIComponent(JSON))。base64 等の具象をここに閉じ込める(DIP) |
| adapter/persistence | `saveImagePayload`(純粋関数) | セーブコード⇄バイト列(`MAGIC 'PWS1'`+長さ+UTF-8)。DOM 非依存=単体テスト可 |
| adapter/persistence | `PngSaveImageCodec`(`SaveImageCodec` ポート実装) | セーブコード⇄PNG。データ画素+アプリ名/保存日時のキャプション帯。canvas 依存をここに閉じ込める |
| main / index.html | セーブ/ロードパネル | クリップボード(コピー/貼り付け)+**画像で保存/画像から読込**。イベント中・パネル表示中は見回し以外ロック |

### 画像トランスポート(同じセーブコードを運ぶ別経路 / OCP)
- 「画像で保存」: `imageCodec.encode(snapshotCodec.encode(snapshot), {appName, savedAt})` で PNG dataURL を生成し
  プレビュー `<img>` に表示。利用者は長押し→「写真に追加」で保存できる。
- 「画像から読込」: ファイル選択 → `snapshotCodec.decode(await imageCodec.decode(file))` → `restore`。
- 画像は上部に**データ画素**(1論理画素=6×6 ブロックに RGB 3byte。中心サンプルで軽微な再圧縮/拡大に耐性)、
  下部に**アプリ名・保存日時のキャプション帯**を描く。データ核(pack/unpack)は純粋関数で分離しテスト可能に保つ。
- `SaveService`/`GameSnapshot`/`SnapshotCodec` は不変=画像対応はアダプタ追加のみ(安全に忘れられる)。

- domain は base64 などの**文字列化の実装を知らない**(`SnapshotCodec` ポートのみ)。差し替え可能(DIP)。
- セーブコード(文字列)を不変の契約にすることで、**将来の「画像で保存/ロード」は、同じ文字列を画像へ
  詰める/取り出す新アダプタを足すだけ**で済む(OCP)。`SaveService`/`GameSnapshot`/codec は変更不要=安全に忘れられる。
- ロードは未知ワールドID等で例外(改ざん検知)。UI は例外メッセージを表示する。

## 操作
- 「💾 セーブ」→ パネルを開く。テキスト欄は**空(プレースホルダー)**で、貼り付け(ロード)用。
- 「コピー」→ その時点のセーブコードを生成してクリップボードへコピー(「コードをコピーしました。」)。
  テキスト欄は空のまま(貼り付けを邪魔しない)。クリップボード API が無い環境は画面外の一時欄経由でコピー。
- コードを貼り付け→「ロード」で復元(成功でパネルを閉じる/失敗で理由を表示)。
- 「画像で保存」→ セーブ画像を生成・プレビュー表示(長押しで写真に保存)。
- 「画像から読込」→ 写真(画像ファイル)を選んで復元(成功でパネルを閉じる/失敗で理由を表示)。
- イベント中はセーブボタンを隠す。パネル表示中は移動/タップを止める(見回しは可)。

## テスト観点
- SaveService: capture が状態を採取 / 注入 clock・appName を採取 / restore が復元しイベント解除 / ラウンドトリップ / 未知ワールドで例外。
- Base64JsonSnapshotCodec: encode→decode 一致 / アプリ名・保存日時の往復 / 旧 v1 後方互換 / 空白許容 / 接頭辞なし・復号不可・不正内容で例外。
- saveImagePayload: pack→unpack 往復 / マルチバイト / マジック・長さの形 / 末尾パディング許容 / 不正マジック・過大長で例外。
