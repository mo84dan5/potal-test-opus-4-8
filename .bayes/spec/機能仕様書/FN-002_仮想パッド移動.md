# 機能仕様書 FN-002 仮想パッド移動

## 概要
ワンフィンガーの「吸い付き型仮想パッド」で移動・旋回・ダッシュを完結させるタッチ操作。
画面のどこをタッチしてもその位置がパッドの原点になり、押している間ドラッグ方向へ移動し続ける。

## 操作仕様

| 操作 | 挙動 |
| --- | --- |
| タッチ開始(**画面の下半分**) | タッチ位置に仮想パッド(ベース円+ノブ)が出現し、そこが原点になる(上半分から開始したタッチは FN-004 の見回しになる。判定は開始位置のみで行い、ジェスチャ中に境界を跨いでも変わらない) |
| 押したままドラッグ | 原点からのベクトルで視点基準の**平行移動**。上=前進、下=後進、左右=左右移動(斜めも可)。引っ張り量で速度が連続変化(最大歩行速度でクランプ) |
| 左右に倒す | 左右への平行移動(ストレイフ)。**視点は回転しない**(視点回転は2本指見回し FN-004 のみ) |
| 指を離す | **その場で即時停止**(慣性なし、速度ゼロ)。パッドUIは消える |
| 素早くはじいて離す(接触 < 250ms かつ 移動 > 40px) | その方向へダッシュ。停止処理の後にインパルスが適用されるため、はじいた勢いだけが残り慣性で滑ってから減速する |
| 移動中に2本目の指が触れる | **移動は継続**したまま、2本目の指が見回し役(FN-004)になる。移動スティック役の指を離したときだけ即時停止する |
| 見回し中(1本目が上半分開始)に2本目の指が触れる | 2本目の指が**位置によらず移動役**になり、そのタッチ位置に仮想パッドが出る |

## 入力モデル(アダプタ層)

- Pointer Events を使用(マウスでも同一動作)。最初の1本指のみ有効。
- スティック値はベース半径 70px で正規化したベクトル `(x, y) ∈ 単位円`(右・下が正)。
- ノブの表示位置はベース円の半径内にクランプする。

## 移動モデル(ドメイン層)

- スティック値からプレイヤー視点基準の**目標速度** `desiredVelocity` を毎フレーム算出:
  - `方向 = right × x + forward × (−y)`(正規化)、`速さ = |stick| × 最大歩行速度(6 m/s)`
  - デッドゾーン `|stick| < 0.12` では目標速度なし(解除扱い)
- 速度は目標速度へ指数追従: `v += (v_target − v) × (1 − exp(−accel·dt))`(accel = 8)
- **指を離した瞬間は即時停止**: 解放イベントで `v = 0`・目標速度解除(`MovementService.halt`)。
- 目標速度がない間(=ダッシュ直後のみ)は指数減衰: `v ×= exp(−damping·dt)`
- **1本指は移動のみ**: スティック操作で視点(ヨー・ピッチ)は一切変化しない。全方向とも視点基準の平行移動である(視点回転は FN-004 の2本指見回しに完全分離)。
- ダッシュはスワイプベクトルを視点基準方向へ変換し `min(maxSpeed, |swipe| × 0.05)` の速度インパルス。
- ポータル通過時(FN-003)は速度に加えて目標速度ベクトルも同じ回転で写像する。
