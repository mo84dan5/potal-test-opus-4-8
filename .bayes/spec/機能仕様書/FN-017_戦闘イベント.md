# FN-017 戦闘イベント

## 概要
NPCとの会話の最後に出る「戦う?【はい/いいえ】」で**はい**を選ぶと戦闘が始まる。
戦闘は次の画面を順に進む:
1. **戦闘開始画面(intro)**: 対戦相手の画像(色アバターのプレースホルダ)+ 意気込みの一言 + 戦う地形名。
2. **キャラ選択画面(select)**: 9体から**メイン**と**サポート**を1体ずつ選ぶ。
3. **戦闘画面(fight)**: 1対1の**リアルタイムアクションバトル**(後述)。
4. **勝敗宣言(result)**: 勝利は「やったー!」の明るい演出、敗北は「どんより」の暗い演出。
5. **戦闘終了画面(outro)**: 相手の一言(勝利/敗北パターン)→「もどる」で**元の世界へ復帰**。

## アクションバトル(fight)
- 各キャラは**3種類の技**を持ち、HPを削り合う。操作は**普段の3人称移動と同じ仮想スティック**を使う:
  - **移動**: 下半分のスティックドラッグ(上=相手へ前進 / 下=後退 / 左右=横移動)。
  - **フリック(はじき)**: 通常のダッシュの代わりに、**上=技1 / 右=技2 / 左=技3**、**下=相手から離れる回避ダッシュ**。
- **ジャスト回避**: 相手の技が当たる直前にダッシュすると無敵(iフレーム)になり被弾しない。
  「ダッシュ中は無敵」という一規則だけで表現する。
- 技は**発生(windup)で予備動作を見せ**(相手に回避の猶予)、終端で命中判定する。当たれば damage、
  射程外なら空振り、相手が無敵なら「ジャスト回避!」。命中後は硬直(recovery)。
- **クールダウン(リードタイム)**: 各技・回避ダッシュには再使用までの待ち時間がある。発動時点から計測し、
  明けるまで同じ技/回避は出せない(連打不可)。HUDのチップで残量(暗→明)を表示する。
- **長距離攻撃(ranged)**: 飛び道具の技(`Technique.ranged`、長い range)。着弾まで**ビーム+着弾バースト**を描く。
- **パーティクルエフェクト**: 技の**発動(cast)・命中(hit)・ジャスト回避(dodge)**で粒子を出し、技が出たことが分かる。
- サポートキャラ編成時はメインのHPにボーナス。
- 敵AIは間合いを詰め、間合い内で技を循環使用(決定的)。プレイヤーの回避を主役にするため敵は回避しない。
- 描画は**専用の three.js シーンによる3D**。カメラは**ステージ(アリーナ)に紐づけた俯瞰**で、
  位置は背面(+Z)の高所に固定しX だけ両者の中点へ部分追従、注視点は中点(画角計算は純粋関数 `computeBattleCamera`)。
  床(円形アリーナ)に2体のアバターを置き、
  技の予備動作は足元の射程リング、ダッシュ無敵は発光、被弾はフラッシュで表現する。
  シミュレーション(CombatService)は描画非依存なので、ビュー(`BattleArenaView`)を差し替えるだけで2D/3Dを切替できる。

戦闘は**副作用ゼロのオーバーレイ**。ワールドからは離れず、終了で重ねた画面を閉じるだけなので、
**位置・フラグ・進捗はすべて元通り**。

## 設計(SOLID / 安全に忘れられるクリーンアーキ)
戦闘は既存のスクリプトイベント(FN-015: ワールド内の自動移動)とは**別概念**として分離する(SRP)。

| 層 | 要素 | 責務 |
| --- | --- | --- |
| domain/values | `Battle`(`BattlePhase`/`BattleOutcome`/`BattleCharacter`/`Opponent`/`BattleDefinition`/HP・攻撃力定数) | 戦闘データの形(直列化可能・純粋) |
| domain/values | `Choice`(`ChoicePrompt`/`ChoiceOption`) | 汎用の選択肢(はい/いいえ)。将来の分岐会話にも再利用(OCP) |
| domain/entities | `BattleSession` | 進行中の戦闘状態(phase/編成/結果)。実HPはアクション戦闘側が持つ |
| domain/services | `BattleService` | 画面フェーズ遷移ステートマシン(純粋)。`finishFight` で決着を受ける。**ワールド状態に一切触れない** |
| domain/values | `Combat`(Technique/CombatFighter/CombatInput/技トリオ/定数) | アクション戦闘のデータ(純粋値) |
| domain/entities | `CombatArena`/`CombatActor` | リアルタイム戦闘の状態(位置・HP・行動状態) |
| domain/services | `CombatService`(+`EnemyController`ポート/`SimpleEnemyController`) | 実時間シミュレーション(移動・技・命中・回避・**クールダウン**)。**エフェクト事象(`CombatArena.effects`)を出すだけ**で描画は知らない。純粋・決定的・**ワールド非依存** |
| adapter/ui | `BattleArenaView`(ポート)+ `BattleArena3DAdapter`(実装) | 戦闘画面の描画/入力。**専用の three.js シーン(三人称カメラ)で3D描画**。`effects` を drain して `BattleParticles` で粒子化、クールダウンHUD表示。**普段の`VirtualStickInputAdapter`を再利用**。決着で onEnd。2D/3D は `BattleArenaView` で差し替え可能(DIP) |
| adapter/ui | `BattleParticles` | 小球プールのパーティクル群(描画専用・SRP)。バースト/ビーム/更新/破棄。乱数は描画側のみ |
| domain/entities | `Interactable.choiceOnEnd` | 会話の最後に提示する選択肢 |
| application | `TapInteractUseCase` | 会話を最後まで送ったとき `choiceOnEnd` を `session.choice` に提示 |
| adapter/ui | `ChoiceOverlayAdapter` | 選択肢のDOM描画。選んだ `value` を返すだけ(振る舞いは知らない) |
| adapter/ui | `BattleOverlayAdapter` | 各画面のDOM描画。ボタンで `BattleService` を駆動し再描画。outro で onExit |
| config | `BATTLES` レジストリ / `NpcSpec.battleId` | 戦闘定義と、戦闘を持つNPCの宣言 |
| main | 配線 | 選択肢の `battle:<id>` を解釈して戦闘開始。戦闘/選択中は世界を凍結・操作ロック |

- 集約 `GameSession` への seam は `activeBattle: BattleSession | null` と `choice: ChoicePrompt | null` の**2つだけ**。
  非戦闘コードは戦闘を**知らないまま**動く(安全に忘れられる)。
- **「進捗・フラグは元通り」は `BattleService` がワールド状態に触れない設計で構造的に担保**される。
- 選択肢の `value`(例 `battle:day-duel`)は不透明文字列で、解釈(戦闘開始)はアプリ層(main)が担う。
  ドメイン/UIは「選んだら何をするか」を知らない(DIP/関心の分離)。
- ロード時(SaveService.restore)は `activeBattle`/`choice` を解除して整合状態へ。

## 操作
- 戦闘NPCに近づき会話→最後まで送ると「戦う?(はい/いいえ)」。**はい**で戦闘開始、**いいえ**で閉じる。
- intro「戦う」→ select。9体をタップでメイン→サポートの順に選択(再タップで解除、同一キャラは不可)。
- 「この編成で戦う」(両方選択時に有効)→ fight。スティックで動き、フリックで技・回避ダッシュ。どちらかのHPが0で決着 → result。
- result「つぎへ」→ outro。outro「もどる」→ 元の世界へ(activeBattle 解除)。
- 戦闘/選択肢の表示中は移動・タップ・セーブを停止(世界を凍結)。

## テスト観点
- BattleService: intro→select / 編成規則(同一キャラ不可・再選択で解除)/ 編成完了で fight /
  finishFight で result / result→outro→isFinished。
- CombatService: 技の発生→命中でHP減 / 射程外は当たらない / ダッシュ無敵中の被弾はジャスト回避(ノーダメージ)/
  ダッシュは離脱方向 / HP0で勝利 / 決着後は進行停止 / 前進入力で接近 /
  **技・回避のクールダウンで連打不可・経過で再使用可** / **発動・命中でエフェクト発行** / **長距離技は遠くても命中**。
- TapInteractUseCase: 会話を最後まで送ると `choiceOnEnd` が `session.choice` に提示される / 無ければ提示しない。
