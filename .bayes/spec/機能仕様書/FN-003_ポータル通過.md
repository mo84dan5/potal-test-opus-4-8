# 機能仕様書 FN-003 ポータル通過

## 概要
プレイヤーがポータル面を横切った瞬間に、暗転やロードなしで接続先ワールドへシームレスに移動する。ポータル面には事前に向こう側が描画されている(FN-001)ため、通過の前後で見えている景色が連続する。

## ポータルの一般化

- ポータルは `id` で識別され、`targetWorldId` + `targetPortalId` で**接続先ワールドの特定ポータル**と対をなす。
- 1つのワールドは複数のポータル(`World.portals`)を持てる。毎フレーム現在ワールドの全ポータルを走査し、最初に横切ったポータルで通過処理を行う。

## 交差判定(ドメイン)

- ポータルは位置 `pos`、ヨー角 `yaw`、半幅 `halfWidth`、高さ `height` を持つ垂直な面。法線 `n = (sin yaw, 0, cos yaw)`。
- フレーム前後のプレイヤー位置 `p₀ → p₁` について:
  1. 符号付き距離 `d₀ = n·(p₀ − pos)`, `d₁ = n·(p₁ − pos)` の符号が反転している
  2. 交差点のポータル接線方向オフセットが `|t| ≤ halfWidth`
  を満たすとき「通過」と判定する。

## 通過時の座標変換

入口ポータルA(現在ワールド)→ 出口ポータルB(`A.targetPortalId` で特定):

```
local  = Rot(−yawA) × (p − posA)
local' = (−local.x, local.y, −local.z)   // Y軸180°反転(表から入り表から出る)
p'     = Rot(yawB) × local' + posB
```

- 速度ベクトルも同じ回転(平行移動なし)で写像する。
- 視点ヨー角: `yaw' = yaw + (yawB − yawA + π)`
- 変換後、`currentWorld` を接続先ワールドに切り替える。
