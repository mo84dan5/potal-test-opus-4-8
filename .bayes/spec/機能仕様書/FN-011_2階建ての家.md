# FN-011 2階建ての家(階段で登れる2階)

## 概要
昼の世界の小屋(扉)から入る室内ワールド「2階建ての家」(`two-floor-house`)。
室内の**右側の階段**を昇ると**ロフト(2階)**へ上がれる。

## 仕組み(多層の高さ場による昇降)
本エンジンはプレイヤーの足元 y を毎フレーム床へスナップする(FN-008)。室内には多層床
`TwoFloorField`(HeightField)を割り当てる。各 (x,z) は複数の床面を持つ:

- **1階**: `h = 0`(常に存在。**ロフトの下にも床がある**)
- **階段**: 右レーン `x ∈ [stairXMin, width/2]`、`z ∈ [stairZBottom, stairZTop]` で `0 → floorHeight` に線形傾斜
- **ロフト(2階)**: `z >= loftFrontZ` で `h = floorHeight`(部屋の奥側半分)

`floorAt(x,z,currentY)` は、現在の足元高さ `currentY + STEP_UP(0.6m)` 以下で**最も高い床面**を返す。
これにより:
- 1階(`currentY≈0`)ではロフト領域でも床は0 → **ロフトの下を歩ける**。
- 階段でだけ `currentY` が徐々に上がり、ロフト(`currentY≈floorHeight`)では2階に留まる。
- `heightAt`(NPC初期化・配置用)は最も高い床面を返す(ロフトのNPCは2階に立つ)。

寸法(`TWO_FLOOR`): 部屋 14×14×6.2m、`floorHeight=3.0`、`loftFrontZ=0`、`stairXMin=3.5`、階段 `z=-4→0`。

## 段差・落下と手すり
コライダーは XZ 円柱のみ(y 無視, FN-006)。高さ y は `MovementService.floorY` が解決する:
- 床が `currentY` 付近(`FALL_STEP_THRESHOLD=0.6m` 以内)なら即スナップ(上り坂・階段・小段差)。
- それを超える落差(ロフトの縁を踏み外すなど)は `FALL_SPEED=14m/s` で**滑らかに降下**(瞬間ワープしない)。

手すりは**階段の開放側(`x=stairXMin`)のみ**。**ロフト前縁には手すりを置かない**(開放型のメザニン)
ことで、1階のプレイヤーがロフト下へ自由に入れる。`twoFloorRailingColliderSpots()` が階段脇の支柱を返す。
部屋の外周は `roomWallColliderSpots`(玄関の扉開口つき)で囲う。

## 入退室
- 入室: 昼の世界の小屋(`portalHouses` の1棟, (-16,-2))の扉 `day-twofloor` をタップ(FN-010 と同じ扉=タップ入室)。
  入室時は玄関扉の正面=1階(h=0)に立つ。
- 退室: 1階に降りて玄関の扉 `twofloor-day` をタップ → 昼の世界へ戻る。

## 描画(`buildTwoFloorInterior`)
1階の床・天井・四方の壁(高さ6.2m)、右側の階段(12段)、奥のロフト床スラブ(上面=floorHeight)、
手すり(横バー+支柱)、家具(1階のテーブル/2階のベッド)、暖色の点光源2灯(各階)。

## NPC
2階(ロフト)に住人NPCを配置(`x=-3, z=4` → 高さ場で `h=floorHeight` に乗る)。階段を昇ると会える。

## テスト観点
- `TwoFloorField`: 1階=0 / ロフト=floorHeight / 階段が単調増加・上下端で連続。
- `twoFloorRailingColliderSpots`: ロフト前縁は階段開口を除いて並ぶ / 階段開放側に支柱。
- `two-floor-house` が interior・floorKind='two-floor'・room を持つ / 昼に扉と小屋2棟。
