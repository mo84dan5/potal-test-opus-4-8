# 機能仕様書 FN-007 案内NPC

## 概要
各ワールドに人型NPCが2人いる: ぶらぶら歩き回る「案内人」と、その場に立っている**静止NPC**。
どちらも近づくと吹き出しで挨拶し、タップすると話をしてくれる。**話しかけている間、相手のNPCは立ち止まる**(ウィンドウを閉じると再開)。

## 徘徊(ドメイン: NpcWanderService)

| 項目 | 内容 |
| --- | --- |
| 行動パターン | スポーン地点を中心とした半径5mの円内のランダムな目的地へ歩く → 到着したら1〜3.5秒立ち止まる → 次の目的地へ |
| 歩行速度 | 1.1 m/s。進行方向を向いて歩く |
| 乱数 | シード付き(ワールドID+個体番号から導出)で決定的。リロードしても同じ経路 |
| 障害物 | 木・石などのコライダーに重なったら押し出される(自分のコライダーは除外) |
| **行動範囲のハード制限** | 押し出し等で徘徊円の外に出されても、毎フレーム境界へ引き戻される(徘徊円の外には**一切出ない**ことを保証。範囲と重なり解消が両立しない場合は範囲を優先) |
| 当たり判定 | NPC自身も半径0.45mのコライダーを持ち、プレイヤーはすり抜けられない(コライダーは移動に追従) |
| 更新 | 毎フレーム全ワールドのNPCを更新(ポータル越しに見える姿も動く) |
| **静止NPC** | `wanderRadius: 0` 指定で徘徊しない。広場の中心を向いて立つ |
| **会話中の停止** | タップで会話を開くと相手が `GameSession.dialogueSpeaker` に記録され、その間は徘徊更新をスキップ。閉じると再開 |
| **会話中の注視** | 会話中の相手NPCは毎フレーム、プレイヤーの方向へ向きを更新する(回り込んでも常にこちらを見る) |
| **距離による自動終了** | 会話中に相手から 6.5m を超えて離れるとウィンドウが自動で閉じ、NPCは徘徊を再開する(FN-005と共通) |

## 会話(FN-005 の機構を流用)

NPCは「動くInteractable」(Npc は Interactable のサブクラス、position を毎フレーム更新)。

| ワールド | 案内人(徘徊) | 静止NPC |
| --- | --- | --- |
| 昼の世界 | こんにちは! / 世界の紹介・ヒント・門の行き先 | **門番**(門の番人・行き来の案内) |
| 夜の世界 | こんばんは! / 世界の紹介・ヒント・門の行き先 | **星読み**(星の話) |
| 雪の世界 | さむいねえ! / 世界の紹介・噂・門の行き先 | **旅商人**(氷柱のかけら売り) |
| 黄昏の遺跡 | ようこそ! / 世界の紹介・いわれ・門の行き先 | **学者**(柱と夕日の研究) |

文言・スポーン位置・服の色・静止指定は `src/config/worldContent.ts` の `WorldDef.npcs`(配列)に集約。

## 見た目(レンダラ)
- 胴体・足(円柱)+頭(球)+とんがり帽子(円錐)+つばの人型。服の色はワールドごとに異なる。
- 吹き出しアンカー高さは 2.0m(頭上)。

## 担当レイヤ
- エンティティ: `src/domain/entities/Npc.ts`(Interactable のサブクラス、徘徊状態・コライダー保持)
- 徘徊: `src/domain/services/NpcWanderService.ts`
- フレーム組込み: `src/application/usecases/TickUseCase.ts`
- メッシュ・毎フレーム反映: `src/adapters/rendering/ThreeRendererAdapter.ts`(buildNpcMesh / syncNpcs)
- 定義: `src/config/worldContent.ts`(NpcSpec)
