# 機能仕様書 FN-006 当たり判定

## 概要
ワールド内のオブジェクトに衝突判定を設け、プレイヤーが通り抜けられないようにする。

## 衝突モデル

| 項目 | 内容 |
| --- | --- |
| コライダー形状 | XZ平面の円(無限高さの円柱)。`Collider = { position, radius }` |
| プレイヤー半径 | 0.35 m |
| 解決方法 | めり込み量を法線方向へ押し出す。複数コライダーの干渉に備え最大3回反復 |
| 壁ずり(スライド) | 障害物へ向かう速度成分のみ打ち消し、接線成分は保存(壁に沿って滑らかに移動できる) |
| 実行順序 | 移動積分 → **衝突解決** → ポータル交差判定(押し戻されたフレームの誤通過を防ぐ) |

## コライダー設定(src/config/worldContent.ts に集約)

| 対象 | 半径 [m] |
| --- | --- |
| 木 | 0.5(幹) |
| 石 | 0.55〜0.95(見た目サイズに応じて) |
| クリスタル | 0.55 |
| ポータル枠の左右柱 | 0.25(×2本) |
| **ポータル面** | **コライダーなし**(FN-003 のシームレス通過を維持) |

## 担当レイヤ
- コライダー値: `src/domain/values/Collider.ts`
- 衝突解決: `src/domain/services/CollisionService.ts`(押し出し+壁ずり)
- フレーム編成: `src/application/usecases/TickUseCase.ts`(移動→衝突→ポータルの順)
- コライダー構築: `src/main.ts`(worldContent のオブジェクト+ポータル柱)

## 制約
- フレームあたりの移動量は dt クランプ(最大1/30s × 最大速度10m/s = 0.33m)によりコライダー半径より十分小さく、すり抜け(トンネリング)は発生しない。
