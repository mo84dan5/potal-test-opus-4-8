# 機能一覧

| 機能ID | 機能名 | 概要 | 関連画面 | 仕様書 |
| --- | --- | --- | --- | --- |
| FN-001 | ポータル描画 | 4つの3Dシーンを保持し、各ポータル面に「向こう側のシーン」をリアルタイム描画する(RTプール + スクリーン空間UV)。ポータルはID対で接続され1ワールドに複数置ける | SCR-001 | [FN-001_ポータル描画](機能仕様書/FN-001_ポータル描画.md) |
| FN-002 | 仮想パッド移動 | **画面の下半分**をタッチするとその位置に仮想パッドが現れ、押している間ドラッグ方向へ視点基準で平行移動し続ける(上=前進、下=後進、左右=左右移動)。視点は回転しない。素早くはじくとダッシュ | SCR-001 | [FN-002_仮想パッド移動](機能仕様書/FN-002_仮想パッド移動.md) |
| FN-003 | ポータル通過 | ポータル面を横切った瞬間に座標・速度・視点を targetPortalId で特定した接続先ポータル系へ写像し、シームレスにワールドを切り替える(全ポータルを走査) | SCR-001 | [FN-003_ポータル通過](機能仕様書/FN-003_ポータル通過.md) |
| FN-004 | 見回し | **画面の上半分の1本指スワイプ**または2本指スワイプで視点を回転。左右=ヨー、上下=ピッチ(±0.9radクランプ)。2本目タッチで移動は即時停止 | SCR-001 | [FN-004_見回し](機能仕様書/FN-004_見回し.md) |
| FN-005 | オブジェクトインタラクション | オブジェクトに近づく(5m)と吹き出し(「これは木です」等)、近く(5.25m)でタップするとメッセージウィンドウが開きタップでコメント送り→最後に閉じる(6.5m超離れると自動で閉じる) | SCR-001 | [FN-005_オブジェクトインタラクション](機能仕様書/FN-005_オブジェクトインタラクション.md) |
| FN-006 | 当たり判定 | 木・石・クリスタル・ポータル枠柱・NPCに円柱コライダーを設定し通り抜け不可(押し出し+壁ずり)。ポータル面は通過可能なまま | SCR-001 | [FN-006_当たり判定](機能仕様書/FN-006_当たり判定.md) |
| FN-007 | 案内NPC | 各ワールドに徘徊する案内人と静止NPC(門番・星読み等)がいて、近づくと挨拶の吹き出し、タップで話をしてくれる。会話中の相手は立ち止まる | SCR-001 | [FN-007_案内NPC](機能仕様書/FN-007_案内NPC.md) |
| FN-008 | 地形起伏 | 地面になだらかな丘があり、プレイヤー・NPCは地形に沿って上り下りする。ポータル周辺とスポーンは平坦化して通過の整合を維持 | SCR-001 | [FN-008_地形起伏](機能仕様書/FN-008_地形起伏.md) |
| FN-009 | 家 | 各シーン(昼・夜・雪・遺跡)に出入りできる家。ドア開口から入れ、中にテレビ(カラーバー)・テーブル、左右に窓(半透明ガラス)。昼の家には住人NPC | SCR-001 | [FN-009_家](機能仕様書/FN-009_家.md) |
| FN-010 | 室内ワールドの家 | 昼の世界の小屋のドアがポータルになっており、くぐると外見より遥かに広い別シーン「大広間」(室内ワールド)へ飛ぶ(TARDIS型)。玄関の扉でもとのシーンへ戻れる | SCR-001 | [FN-010_室内ワールドと家配置](機能仕様書/FN-010_室内ワールドと家配置.md) |
| FN-011 | 2階建ての家 | 昼の世界の小屋の扉から入る室内「2階建ての家」。右側の階段を昇るとロフト(2階)に上がれる。床の高さは高さ場で表現し、段差の縁は手すりで転落を防ぐ。2階に住人NPC | SCR-001 | [FN-011_2階建ての家](機能仕様書/FN-011_2階建ての家.md) |
| FN-012 | 落下と滑空 | 高い段差を踏み外すと一定速度で落下する。落下中にタップ or 2本指の2本目タップで「滑空状態」へ移行し、降下がさらに遅くなりゆっくり前後左右に移動できる。着地で解除 | SCR-001 | [FN-012_落下と滑空](機能仕様書/FN-012_落下と滑空.md) |
| FN-013 | 崖とよじ登り | 各世界に崖(メサ)があり、ほぼ垂直の壁を時間をかけてよじ登れる。頂上は平らで、踏み外すと落下・滑空できる。緩斜面・階段は従来どおり即登れる | SCR-001 | [FN-013_崖とよじ登り](機能仕様書/FN-013_崖とよじ登り.md) |
| FN-014 | 視点モード | 1人称/3人称をHUDボタンで切替(**既定は3人称**)。3人称はカメラを後方上方に置きプレイヤーのアバターを表示する。移動・当たり判定等の挙動は不変 | SCR-001 | [FN-014_視点モード](機能仕様書/FN-014_視点モード.md) |
| FN-015 | イベント | NPC等をタップで始まるスクリプト化イベント(say/walkTo/escort/moveProp/wait/setFlag)。実際に歩いて案内、岩が動いて道が開く等。フラグ・条件で分岐。イベント中は見回し以外の操作不能 | SCR-001 | [FN-015_イベント](機能仕様書/FN-015_イベント.md) |
| FN-016 | セーブ/ロード | ゲーム状態(ワールド/姿勢/フラグ/完了イベント/プロップ位置)を文字列のセーブコードで保存・復元。コピー&貼り付けに加え、アプリ名・保存日時つきの画像(PNG)で保存/写真から読込も可能(snapshot⇄文字列の境界に画像トランスポートを追加) | SCR-001 | [FN-016_セーブロード](機能仕様書/FN-016_セーブロード.md) |
| FN-017 | 戦闘イベント | NPC会話の「戦闘しますか【はい/いいえ】」から始まる。戦闘開始画面(相手画像・意気込み・地形)→キャラ選択(9体からメイン+サポート)→**1対1アクションバトル**→勝敗宣言→終了画面(相手の一言)→元の世界へ復帰。戦闘は普段の3人称と同じ仮想スティックで移動、フリックで技(上右左=3種の技)・ダッシュ(下=離脱)。相手の技直前のダッシュでジャスト回避。技/回避には再使用クールダウン、技の発動・命中はパーティクルエフェクト、長距離攻撃(飛び道具)もある。専用3Dシーンで描画。戦闘は副作用ゼロのオーバーレイで進捗・フラグは不変 | SCR-001 | [FN-017_戦闘イベント](機能仕様書/FN-017_戦闘イベント.md) |

| FN-018 | 手書き風シェーダー | 世界の最終描画にフルスクリーンの手書き(スケッチ/水彩)風ポストエフェクトをかける。輝度Sobelによるインク輪郭線、紙の色み+粒状ノイズ、約8fpsの手書きゆれ(boil)、軽い色の平坦化。本描画をオフスクリーンに描き SketchPass で画面へ出力(既存パイプラインは不変・外せば従来描画に戻る) | SCR-001 | [FN-018_手書き風シェーダー](機能仕様書/FN-018_手書き風シェーダー.md) |

## 非機能要件

| 項目 | 内容 |
| --- | --- |
| 対応環境 | モバイルブラウザ(iOS Safari / Android Chrome)、PCブラウザ(マウス操作でも動作) |
| 配信方式 | GitHub Pages による静的配信(サーバサイドなし) |
| パフォーマンス | devicePixelRatio 上限2、ポータル用レンダーターゲットは画面解像度に追従 |
| 描画品質 | Zファイティング対策: カメラ near=0.2/far=200 で深度精度を確保。面の重なりを避ける(崖は地面と二重に描かずフラスタムのみ、階段の段は重ねて同一平面の接触を回避)。ポータルは自作シェーダのため対数深度バッファは不使用 |
| アーキテクチャ | クリーンアーキテクチャ(domain は three.js / DOM 非依存。依存方向は常に内側へ) |
