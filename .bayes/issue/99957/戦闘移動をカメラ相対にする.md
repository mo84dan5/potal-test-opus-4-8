## Issue Workerの計画

### [o] セクション0 このIssueで何をやりたいか

**要件**: 戦闘の移動方向を**カメラ相対**にする(スティック上=画面奥、右=画面右…)。
現状は「前=相手方向 / 左右=相手方向の直交」で、固定俯瞰カメラだと操作感がねじれる。

**設計方針(SOLID / 安全に忘れられる)**:
- 「カメラ相対」への変換は**カメラを知っているビュー(アダプタ)の責務**。
  → `CombatInput` を **ワールド空間の移動ベクトル `{moveX, moveZ}`** に変更し、ドメインはそれを適用するだけ。
  ドメイン(`CombatService`)はカメラを一切知らないまま(関心の分離・DIP)。
- アダプタは毎フレーム、カメラの水平基底(前方/右)とスティックから `moveX/moveZ` を解決して渡す。
- 技の照準・離脱ダッシュ方向は従来どおり「相手方向」基準(攻撃は相手に向く)。移動だけカメラ相対に。
- 敵AIも `moveX/moveZ`(相手へ向かうワールド方向)を返すよう更新。

### [o] セクション3 実装・テスト・デプロイ

#### 方針
- 移動入力をワールド空間ベクトルに統一。カメラ相対変換はアダプタが担いドメインは非依存。

#### 結果
- Combat.ts: `CombatInput {moveX, moveZ, action}`、`IDLE_INPUT` 更新。
- CombatService.ts: idle 移動は moveX/moveZ を(長さ1にクランプして)MOVE_SPEED で適用。facing/dash は相手基準を維持。
  SimpleEnemyController は相手へ向かうワールド方向を返す。
- BattleArena3DAdapter.ts: カメラの水平 forward/right を用い `moveX = right.x*sx + fwd.x*(-sy)` 等で解決。
- テスト: flick を moveX/moveZ に、移動テストは「相手方向へのワールド移動で接近」を確認 → vitest 全パス。ビルド成功。
- 仕様: FN-017 の移動記述を更新。
- デプロイ: main push → GitHub Pages。
