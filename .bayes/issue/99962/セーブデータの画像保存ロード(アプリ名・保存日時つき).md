## Issue Workerの計画

### [o] セクション0 このIssueで何をやりたいか

**要件**:
1. セーブ時にセーブデータを**画像(PNG)**として生成し、写真アプリに保存できる。
2. ロード時は**写真をアップロード**(ファイル選択)して復元できる。
3. 画像に**アプリ名・保存日時**を記録する(人が見て分かる + データにも内包)。

**設計の要(境界=文字列セーブコード を不変契約に保つ)**:
- 既存の `GameSnapshot ⇄ 文字列(セーブコード)` の契約は**変えない**。画像は「同じセーブコードを
  運ぶ別トランスポート」として足す(OCP / 安全に忘れる)。`SaveService`/`SnapshotCodec` は不変。
- アプリ名・保存日時は **`GameSnapshot` に追加**する(`appName` / `savedAt`)。これにより
  既存の文字列コーデックを通して**自動的に画像へも内包**され、ロード時もそのまま復元・検証できる。
  - 時刻・アプリ名は外部都合(非決定的)なので、`SaveService` に **`appName` と `Clock`(時刻ポート)を注入**
    する(DIP)。`SaveService` は `new Date()` を直接呼ばない=テスト可能・単一責務を保つ。
- 画像トランスポートは2層に分割(SOLID/テスト容易性):
  - **`saveImagePayload.ts`(純粋関数)**: `packPayload(code)→Uint8Array` / `unpackPayload(bytes)→string`。
    `MAGIC('PWS1') + 長さ(4byte BE) + UTF-8 本体`。DOM 非依存=node でテスト可能。
  - **`PngSaveImageCodec.ts`(アダプタ)**: canvas でバイト列を画素に詰めて PNG dataURL を生成。
    上部にデータ画素、下部に**アプリ名・保存日時のキャプション帯**を描画。decode は `<img>`→canvas で
    画素中心をサンプルして `unpackPayload`。canvas/DOM 依存はここに閉じ込める。
- UI は「画像で保存」(ダウンロード/長押し保存用 `<img>` 表示)と「画像から読込」(ファイル選択)を追加。

### [o] セクション3 実装・テスト・デプロイ

#### 方針
- 文字列セーブコードを不変契約に保ち、画像は別トランスポート(OCP)。
- アプリ名・保存日時は `GameSnapshot` に持たせ既存コーデックで自動内包。時刻は `Clock` 注入で決定化。
- 画像コーデックは「純粋なバイト詰め」と「canvas グルー」に分離(テスト可能な核を残す)。
- 旧バージョン(v1, appName/savedAt 無し)コードも読めるよう isValidSnapshot は新フィールドを任意許容。

#### 結果
- domain/values/Clock.ts: `Clock { nowIso(): string }`(時刻ポート)。
- domain/values/GameSnapshot.ts: `appName`/`savedAt` 追加、SNAPSHOT_VERSION=2、isValidSnapshot で
  新フィールドを「string または未定義」として許容(後方互換)。
- domain/services/SaveService.ts: コンストラクタで `appName` と `Clock` を受け取り、capture で
  `appName`/`savedAt`(ISO)を採取。既定値で旧テストとの互換も維持。
- adapters/persistence/saveImagePayload.ts: `packPayload`/`unpackPayload`(MAGIC+長さ+UTF-8)。
- adapters/persistence/PngSaveImageCodec.ts: `SaveImageCodec` ポート + PNG 実装(データ画素 + キャプション帯)。
- main.ts: `new SaveService('Portal Walk', { nowIso: () => new Date().toISOString() })`、
  画像保存(プレビュー表示+ダウンロード)/画像読込(ファイル→decode→restore)を配線。
- index.html: 「画像で保存」「画像から読込」ボタン、隠し `<input type=file>`、プレビュー `<img>`。
- テスト: saveImagePayload(round-trip / 不正 MAGIC / 長さ不足)、SaveService(注入 clock/appName)、
  Codec(新フィールド round-trip) → vitest 全パス。ビルド成功。
- 仕様書: FN-016 に画像トランスポート・メタ情報を追記。
- デプロイ: main へ push → GitHub Pages Actions。
