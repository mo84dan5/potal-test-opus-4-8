## Issue Workerの計画

### [o] セクション0 このIssueで何をやりたいか

**現象**: 2階建ての家(`two-floor-house`)で、ロフト(2階)の**真下の1階**に入れない。

**原因(事実確認済み)**: 2つの要因の合わせ技。
1. `TwoFloorField.heightAt` は `z >= loftFrontZ` で**常に** `floorHeight`(2階)を返すため、
   ロフト領域では足元が必ず2階に持ち上げられ、1階(h=0)に立てない(高さ場が単一値の限界)。
2. ロフト前縁(z=loftFrontZ)に**手すりコライダー**があり、1階のプレイヤーがロフト下へ
   水平移動するのを物理的に塞いでいる。

**方針(多層床+滑らか落下+手すり調整)**:
1. 床を「現在の足元高さ `currentY` を考慮して立つべき面を返す」多層床にする。
   `HeightField` に任意メソッド `floorAt(x,z,currentY)` を追加。`TwoFloorField` は (x,z) の候補面
   {1階0 / 階段ランプ / ロフトH} のうち `currentY + STEP_UP` 以下で最も高い面を返す。
   → 1階(currentY≈0)でロフト領域に入っても2階に吸い上げられず、**ロフト下を歩ける**。
   → 階段でだけ徐々に上がり、2階(currentY≈H)では2階に留まる。
2. `MovementService`/`TickUseCase` の足元スナップを `floorAt` 対応にし、段差を大きく下る時は
   一定速度で**滑らかに落下**(瞬間ワープを避ける)。単一面の地形は従来どおり(完全後方互換)。
3. ロフト前縁の**手すりコライダーを撤去**(1階からロフト下へ通れるように)。階段脇の手すりは残す。
   描画もロフト前縁の手すりを除去(開放型のメザニン)。

### [o] セクション3 実装・テスト・デプロイ

#### 方針
- 単一面の高さ場(屋外・FLAT)は `floorAt` 未実装 → `heightAt` フォールバックで挙動不変(既存テスト維持)。
- `STEP_UP`/`FALL_THRESHOLD` は階段1フレームの上下動より十分大きく、ロフト段差(3m)より小さい値(0.6)。

#### 結果
- Terrain.ts: `HeightField.floorAt?` を追加。`TwoFloorField` を多層化(surfacesAt={1階0/階段/ロフト}、
  heightAt=最大面、floorAt=currentY+STEP_UP 以下で最も高い面)。
- MovementService.ts: `floorY`(段差越えは即スナップ、大段差は FALL_SPEED で滑らか降下)を追加し tick で使用。
  単層(屋外/FLAT)は heightAt フォールバックで挙動不変。
- TickUseCase.ts: 衝突後の再スナップを `movement.floorY(...,0)` に変更。
- worldContent.ts: `twoFloorRailingColliderSpots` をロフト前縁を除き**階段脇のみ**に。
- ThreeRendererAdapter.ts: ロフト前縁の手すり描画を撤去(開放型メザニン)。
- テスト: floorAt の多層判定 / 滑らか降下 / 手すりが階段脇のみ を追加・更新 → vitest 136件 全パス。ビルド成功。
- 仕様書 FN-011 を多層床・落下・手すり方針に更新。
- デプロイ: main へ push → GitHub Pages Actions で自動デプロイ。
