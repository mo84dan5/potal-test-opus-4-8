## Issue Workerの計画

### [o] セクション0 このIssueで何をやりたいか

**要件**: 各世界に崖(高い壁)を作り、**よじ登れる**ようにする。

**事実確認の結論**:
- プレイヤーの y は毎フレーム `terrain.heightAt`(多層なら `floorAt`)へスナップ。現状、上りは段差に関係なく
  即スナップ(=どんな高さもワープして登れる)。地面メッシュは `terrain.heightAt` を頂点変位で描画。
- 落下/滑空(FN-012)が既にあり、崖から飛び降りる動線に再利用できる。

**方針(高さ場のメサ + 登坂レート制限)**:
- 崖 = 高さ場の**メサ**(平らな頂上 + 急斜面の側面)。`CliffField`(HeightField)で各世界の地形に合成。
- `MovementService.tick` の垂直解決を3分岐に:
  - 急な上り(`delta = 床 - 現在y > FALL_STEP_THRESHOLD`)= **よじ登り**: 1フレームの上昇を `CLIMB_SPEED` に
    制限し、水平移動も同率に抑えて面に沿わせる(`player.climbing=true`)。→ 急斜面は時間をかけて登る。
  - 段差越え(緩斜面・小段差): 即スナップ(従来どおり)。緩い丘は影響を受けない。
  - 大きな落差: 落下(FN-012、滑空可)。
- 崖の頂上から踏み外すと落下→滑空できる(機能連携)。
- レンダラは地面メッシュ(高さ場追従)に加え、崖の四角錐フラスタムメッシュを重ねて見栄えを出す。
- UI に「🧗 よじ登り中」を表示。

### [o] セクション3 実装・テスト・デプロイ

#### 方針
- 登坂レート制限は `delta > FALL_STEP_THRESHOLD(0.6)` の急上昇時のみ。緩斜面/階段は従来どおり(後方互換)。
- 多層床(2階建て)は floorAt が床を抑えるため誤登坂しない。

#### 結果
- Terrain.ts: `CliffSpec`/`CliffField`(メサ: 頂上矩形+ほぼ垂直の壁、heightAt=max(地形,メサ))を追加。
- MovementService.ts: `CLIMB_SPEED=2.5` を追加。tick の垂直解決を「よじ登り(急上昇を制限+水平同率縮小)/
  段差越え/落下」の3分岐に。floorY も急な上りは再スナップで登らないよう調整。
- Player.ts: `climbing` を追加。
- worldContent.ts: `CLIFF`(slopeRun=0.3 のほぼ垂直壁)・`WorldDef.cliff` を追加し、4世界に `cliff:{x:16,z:16}` を配置。
- main.ts: buildTerrain で CliffField を合成。よじ登りUIをトグル。
- ThreeRendererAdapter.ts: `buildCliff`(四角錐フラスタムの岩塊)を追加。
- index.html: 「🧗 よじ登り中」インジケータを追加。
- テスト: CliffField・よじ登り(上昇制限/頂上到達/緩斜面は即登る)を追加 → vitest 154件 全パス。ビルド成功。
- 仕様書: FN-013 新規、機能一覧/index に追記。
- デプロイ: main へ push → GitHub Pages Actions で自動デプロイ。
