## Issue Workerの計画

### [o] セクション0 このIssueで何をやりたいか

**現象**: 2階建ての家で、階段の**裏(下)**の空間に1階(地面レベル)から侵入できる。
階段はソリッドな箱で組まれているため、裏から入ると階段メッシュの中に潜り込んでしまう。

**原因(事実確認済み)**: #99975 でロフト前縁の手すりを撤去し、多層床で1階に立てるようにした結果、
1階のプレイヤーがロフト前縁(z=loftFrontZ)側=**階段の裏**から階段フットプリント
(x∈[stairXMin,w], z∈[stairZBottom,0])の地面レベルへ入り込めるようになった。
コライダーは XZ 円柱(y 無視)のため、階段フットプリントを塞ぐ単純なコライダーは
**階段を登る人や2階(ロフト)も巻き込んで**ブロックしてしまう(ユーザー指摘の「2階に影響」)。

**方針(高さ制限コライダー)**:
1. `Collider` に任意の `yMin`/`yMax`(高さ範囲)を追加。`CollisionService` はプレイヤーの
   足元〜頭(feet〜feet+PLAYER_HEIGHT)が範囲と重なる時だけ衝突解決する。
   未指定は従来どおり無限高さ(全床に影響=壁など)。
2. 階段の裏側(z=loftFrontZ, x∈[stairXMin,w])と開放側(x=stairXMin, z∈[stairZBottom,0])に
   **地面レベル限定(yMax < floorHeight)**のコライダーを置く。
   → 1階の侵入を塞ぎつつ、ロフト(足元≈floorHeight)には影響しない。

### [o] セクション3 実装・テスト・デプロイ

#### 方針
- 高さ制限は任意。既存コライダー(壁・木・ポータル枠など)は yMin/yMax 未指定で従来挙動(後方互換)。
- STAIR_BLOCK_YMAX は floorHeight 未満(例 2.6)。1階(足元0)を塞ぎ、ロフト(足元3)に影響しない。

#### 結果
- Collider.ts: 任意の `yMin`/`yMax`(高さ範囲)を追加。
- CollisionService.ts: `PLAYER_HEIGHT=1.7` を追加し、プレイヤーの足元〜頭が高さ範囲と重なる時だけ
  押し出すように変更(未指定は無限高さ=従来どおり)。
- worldContent.ts: `twoFloorRailingColliderSpots` を `twoFloorStairBlockerSpots`(開放側+裏側)に置換。
  `TWO_FLOOR_STAIR_BLOCKER_YMAX = floorHeight-0.4(=2.6)` を追加。
- main.ts: 階段ブロッカーを `yMin=0/yMax=2.6` の高さ制限コライダーとして組立。
- テスト: 高さ制限コライダー(1階で効き2階で無効/未指定は全高さ)、ブロッカー位置(開放側・裏側・前縁は空ける)
  を追加・更新 → vitest 141件 全パス。ビルド成功。
- 仕様書 FN-011 に高さ制限コライダーと階段裏侵入防止を追記。
- デプロイ: main へ push → GitHub Pages Actions で自動デプロイ。
