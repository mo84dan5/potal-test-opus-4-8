## Issue Workerの計画

### [o] セクション0 このIssueで何をやりたいか

**要件**:
1. **リードタイム(クールダウン)**: 各技と回避ダッシュに「再使用できるまでの待ち時間」を付ける。
2. **パーティクルエフェクト**: 技が発動・命中・回避したことが見て分かるエフェクトを出す。
3. **長距離攻撃**: 飛び道具的な長距離の技を作る(命中位置までビーム/弾のエフェクト)。

**設計の要(ドメインは純粋なまま、描画はアダプタへ)**:
- クールダウンは**純粋ドメイン**で扱う: `Technique.cooldown`(秒)と `CombatActor.techCd[]`/`dashCd`。
  `CombatService` がフレーム毎に減算し、技/ダッシュ発動時に確認・セットする(決定的=テスト可能)。
- エフェクトは**描画専用の関心事**なので、ドメインは「何が起きたか」のデータだけを出す:
  `CombatArena.effects: CombatEffect[]`(cast/hit/dodge + 位置・方向・色・ranged)。
  `CombatService` が push、**ビュー(3Dアダプタ)が毎フレーム drain して** パーティクルを生成する。
  → ドメインは three.js/パーティクルを知らない(関心の分離・安全に忘れられる)。
- 長距離攻撃は `Technique.ranged` フラグ + 長い `range` で表現。ビューは ranged の命中で
  始点→着弾点のビーム + 着弾バーストを描く。近接技は着弾バーストのみ。
- パーティクル描画は `BattleParticles`(小球プール)に分離(SRP)。乱数は描画側のみ(ドメインは決定的のまま)。

### [o] セクション3 実装・テスト・デプロイ

#### 方針
- クールダウンは純粋ドメインで決定的に。エフェクトはデータ(effects)として出し描画はビューが担う。
- 長距離は ranged フラグ + 長 range。ビューがビーム/弾を描く。
- パーティクルは BattleParticles に分離。乱数は描画側のみ。

#### 結果
- Combat.ts: `Technique.cooldown`(必須)・`ranged?`、`DASH_COOLDOWN`、`CombatEffect`(kind/x,z/from/color/ranged)。
  技プリセットに cooldown を付与、TECH_RANGED を長距離(range6〜7)+ranged に、相手用 TECH_DUELIST(近接+火球)を追加。
- CombatArena.ts: `CombatActor.techCd:[0,0,0]`/`dashCd`、`CombatArena.effects`。
- CombatService.ts: updateActor 先頭でクールダウン減算、技/ダッシュは cd 確認後に発動して cd セット、
  発動(cast)・命中/回避/空振り(hit/dodge)で effects を push。SimpleEnemyController は cd 中の技を避ける。
- BattleParticles.ts: 小球プールの burst/beam/update/dispose。
- BattleArena3DAdapter.ts: 毎フレーム effects を drain してパーティクル生成、クールダウンHUDチップ(技1/2/3/回避)を更新。
- worldContent.ts: 技に cooldown、長距離技 ranged、相手トリオ。
- テスト: クールダウンで連打不可 / 時間経過で再使用可 / 命中・発動で effects 発行 / 長距離(range大)命中 → vitest 全パス。ビルド成功。
- 仕様: FN-017 更新、機能一覧更新。
- デプロイ: main push → GitHub Pages。
