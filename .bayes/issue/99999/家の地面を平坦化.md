## Issue Workerの計画

### [o] セクション0 このIssueで何をやりたいか

家(`day`ワールド, `house: { x: -10, z: -13 }`, 寸法 6×5×2.6m)の足元の地面が起伏で盛り上がり、
床を突き抜けて干渉している。家の占有範囲(フットプリント)では地面を**完全に平ら(高さ0)**にしたい。

**原因(事実確認済み)**:
`FlatSpot` の平坦化は `t = d/radius` の smoothstep で「中心だけ」高さ0、縁に向かって元の起伏へ戻る。
家には `FLAT_HOUSE_RADIUS = 7` のスポットが付くが、家の角(中心から約3.9m)では地形高さが振幅の約59%
(≈0.47m)まで残り、`buildHouse` が床を中心高さ(≈0)に置くため、周囲の地面が床を突き抜ける。

**解決方針**:
`FlatSpot` に「内側は完全平坦」のプラトー半径 `flatRadius` を追加する。
`d <= flatRadius` で高さ0、`flatRadius < d < radius` で smoothstep により元の起伏へ滑らかに復帰。
家にはフットプリントを覆う `flatRadius`(家の半対角 hypot(3,2.5)≈3.9m + 余白)を与える。

### [o] セクション3 実装・テスト・デプロイ

#### 方針
- `FlatSpot.flatRadius`(任意, 既定0)を追加。後方互換(既存スポットは従来通り中心のみ平坦)。
- `heightAt`: `d <= inner` → `h = 0`、`inner < d < radius` → `t=(d-inner)/(radius-inner)` の smoothstep。
- 家: `flatRadius = ceil(hypot(width/2, depth/2)) + 余白` でフットプリント全体を平坦化。外側 `radius=7` で滑らかに地形へ。
- 地面メッシュ(`addGround`)・家配置(`buildHouse`)は同じ高さ場を共有するため、ドメイン側の修正だけで描画も整合。

#### 結果
- Terrain.ts: `FlatSpot` に `flatRadius?` 追加、プラトー対応の smoothstep を実装。
- main.ts: `FLAT_HOUSE_PLATEAU_RADIUS` を導入し家スポットへ付与。
- テスト: プラトー内が完全平坦であること・境界連続性を追加。`vitest run` 全件パス。
- ビルド: `npm run build`(tsc + vite)成功。
- デプロイ: main へ push → GitHub Pages Actions(`.github/workflows/deploy.yml`)で自動デプロイ。
