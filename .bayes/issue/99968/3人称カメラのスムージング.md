## Issue Workerの計画

### [o] セクション0 このIssueで何をやりたいか

**現状**: 3人称の遮蔽回避(#99969)で、カメラと主人公の間に物が入ると**カメラ距離が即座に**手前へ寄り、
物が外れると**即座に**元の距離へ戻る。この瞬間がカクッと感じる。

**要件**: カメラ位置が変わる瞬間にスムージング(なめらかな補間)をかける。

**方針(プレゼンテーションのみ・ドメイン非変更)**:
- 3人称カメラの**ブーム距離**(注視点→カメラの距離)を指数補間でスムージングする。
  - 絶対位置ではなく「距離」を補間するため、ポータル通過でプレイヤーが瞬間移動しても
    カメラが世界を横断して飛ぶことがない(距離は相対量で安定)。
- 補間は純粋関数 `smoothTowards(current, target, rate, dt)` に切り出し単体テスト。
- 1人称は頭に固定のため補間しない(遅延=酔いを避ける)。3人称へ初めて入ったフレームは即スナップで開始。
- `render(dt)` に dt を渡す。

### [o] セクション3 実装・テスト・デプロイ

#### 方針
- 距離(ブーム長)を補間 → 遮蔽の出入りがなめらかに。テレポ・追従には悪影響なし。
- 1人称は補間しない。3人称開始時は即スナップ(smoothedCamDist=null→desired)。

#### 結果
- cameraView.ts: `smoothTowards(current, target, rate, dt)`(フレームレート非依存の指数補間)を追加。
- ThreeRendererAdapter.ts: `smoothedCamDist` を保持し、3人称のブーム距離を `smoothTowards`(rate=12)で補間。
  初回は即スナップ、1人称では null リセット。`render(dt)` で dt を受け、syncCamera へ伝播。
- main.ts: `renderer.render(dt)`。
- テスト: smoothTowards(dt0/部分接近/収束/rate比較)を追加 → vitest 166件 全パス。ビルド成功。
- 仕様書 FN-014 にスムージングを追記。
- デプロイ: main へ push → GitHub Pages Actions で自動デプロイ。
