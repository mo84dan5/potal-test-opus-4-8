## Issue Workerの計画

### [o] セクション0 このIssueで何をやりたいか

**現象**: 画面下半分をドラッグして移動スティックを出すと、iOS Safari がテキスト選択を開始し、
選択ハンドル(青い丸)と選択ボックスがスティック周辺に表示される(スクショ参照)。

**原因(事実確認済み)**:
- `html, body` および `#app`・`canvas` に `user-select: none` / `-webkit-user-select: none` /
  `-webkit-touch-callout: none` が**未指定**。HUD等の一部要素にしか選択禁止が付いていない。
- そのため、canvas 上のタッチドラッグ(=移動操作)で iOS のテキスト選択ジェスチャが発火する。

**方針**:
1. `html, body` にグローバルで選択禁止(`user-select:none` 系 + `-webkit-touch-callout:none` +
   `-webkit-tap-highlight-color: transparent`)を付与(継承で全要素に効く)。スティックUIにも明示。
2. `VirtualStickInputAdapter` の `pointerdown` で `preventDefault()` を呼び、選択/コールアウトの
   既定動作を抑止(古いiOS向けの保険)。

### [o] セクション3 実装・テスト・デプロイ

#### 方針
- CSS が主たる修正(iOS Safari の選択は `-webkit-user-select:none` が正攻法)。preventDefault は保険。
- 既存テストはピュア関数(zoneForTouch/roleForTouch)のみ参照のため影響なし。

#### 結果
- index.html: `html, body`・`#app`・`#app canvas`・`.stick-base/.stick-knob` に
  `user-select:none` / `-webkit-user-select:none` / `-webkit-touch-callout:none`(body には
  `-webkit-tap-highlight-color: transparent` も)を付与。
- VirtualStickInputAdapter.onDown: `if (e.cancelable) e.preventDefault();` を追加。
- 仕様書 FN-002 に選択抑止の注記を追加。
- テスト: 既存123件 全パス(ピュア関数のみ参照のため影響なし)。ビルド成功。
- デプロイ: main へ push → GitHub Pages Actions で自動デプロイ。
