## Issue Workerの計画

### [o] セクション0 このIssueで何をやりたいか

**現象**: #99974 で階段裏侵入を防ぐ階段ブロッカーを `yMax = floorHeight-0.4 = 2.6` で置いた結果、
**階段を上がれなくなった**。

**原因(事実確認済み・計算)**:
階段は `stairZBottom=-4 → stairZTop=0` で `0→floorHeight(3.0)` に傾斜(傾き0.75/m)。
**裏側ブロッカー**は `z=loftFrontZ(0)`、半径0.35。プレイヤー半径0.35を足すと、
プレイヤー中心が `z > -0.7` の範囲で作用する。`z=-0.7` での階段面の高さ =
`(4-0.7)/4 × 3 = 2.475m`。一方ブロッカーの `yMax=2.6 > 2.475` なので、
**階段の上端付近(足元≈2.475)を登るプレイヤーがブロッカーに引っかかり、押し戻されて登り切れない**。

**方針**: 階段ブロッカーの `yMax` を**腰高の低い値(1.2m 程度)**へ下げる。
- 1階(足元0)の侵入は引き続き阻止(`feet 0 <= 1.2` で作用)。
- 階段上端を登る人(足元 2.4〜3.0)・2階(足元3.0)には作用しない(`feet > 1.2`)→ 登坂・2階に影響しない。
- `z=0` で立てる床は「1階(0)」か「ロフト(3.0)」のみで中間が無いため、低い yMax でも侵入は完全に塞げる。

### [o] セクション3 実装・テスト・デプロイ

#### 方針
- ブロッカー機構(高さ制限コライダー)は #99974 のまま。値だけを安全域へ調整。
- 登坂を妨げない条件: `yMax < (裏側ブロッカー作用境界 z=-(0.35+0.35) での階段面の高さ = 2.475)`。
  侵入阻止の条件: `yMax > 0`。→ 1.2 は両条件を満たす安全な腰高。
