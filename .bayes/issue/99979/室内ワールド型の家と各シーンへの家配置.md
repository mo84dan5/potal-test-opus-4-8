## Issue Workerの計画

### [o] セクション0 このIssueで何をやりたいか

**要件**
1. 「中に入ると大きな別シーン(室内)に飛ばされる」タイプの家を新設する(TARDIS型: 外は小さい小屋、入ると広大な室内)。
2. 夜・雪・遺跡の各シーンにも家を配置する(既存の歩いて入れる家=FN-009 と同型)。

**設計方針(事実確認済みアーキテクチャに基づく)**
- 本アプリは「ワールド(=シーン)」を複数持ち、`Portal` の ID 対(`targetWorldId`/`targetPortalId`)で接続。
  ポータル面を横切ると `PortalTraversalService` が座標を写像してシームレスにワールドを切り替える(FN-003)。
  → **「中に入ると別シーン」= 家のドアをポータルにし、専用の室内ワールドへ接続する**のが既存資産を最大限再利用できる最善手。
- 室内ワールドは `interior: true` の新ワールドとして追加。レンダラは床・壁・天井・暖色光で囲まれた**広い部屋**を描画。
  室内には戻り用ポータル(玄関)を1枚置き、外の小屋ドアと対にする。
- 他シーンの家は既存 `house?: HouseSpec` を各 WorldDef に足すだけで、`buildHouse`/`houseInteractables`/
  `houseColliders`/地形平坦化がすべて汎用処理として効く(FN-009 を再利用)。

### [o] セクション3 実装・テスト・デプロイ

#### 方針
- ポータルは既存の双方向ID対方式を踏襲。室内入口/玄関ともに壁にドア開口(コライダーの隙間)を設け、
  ポータル面に到達できるようにする(家のドア開口と同じ手法)。
- 室内ワールドは `terrainAmplitude: 0`(完全平坦)。
- 既存テスト(家コライダー網羅・地形)を壊さないこと。新規にポータル相互接続の整合テストを追加。

#### 結果
- worldContent.ts: `PortalHouseSpec`/`RoomSpec`/`PORTAL_HOUSE` 追加。室内ワールド `grand-hall`(30×24×6m)を追加。
  day に小屋(portalHouse (10,-13))と室内行きポータル `day-grandhall`、夜/雪/遺跡に `house` を追加。
  `portalHouseWallColliderSpots`/`roomWallColliderSpots` を追加。
- main.ts: 小屋の地形平坦化、`portalHouseColliders`/`roomColliders` を buildWorld へ組込み。
- ThreeRendererAdapter.ts: `buildRoom`(囲まれた広間)・`buildPortalHouse`(外観小屋)を追加、buildWorld で室内分岐。
- テスト: ポータル相互接続の整合・ID一意・室内ワールド・各シーンの家・ドア開口/壁網羅を追加 → `vitest` 117件 全パス。
- ビルド: `npm run build`(tsc + vite)成功。
